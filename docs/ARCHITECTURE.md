# Architecture — как игра должна работать

Не «как писать код», а **какие системы есть и что игрок видит/чувствует**.

## Общая схема

```
Игрок → Лаборатория (капсулы, выбор монстра) → UI (выбор цели)
         ↓
       Сервер (проверяет, считает, сохраняет)
         ↓
       Мир (монстр идёт к дому, пакость на месте, уведомления)
         ↓
       UI (награда, тост «твой Гремлин пойман», баланс)
```

Клиент не решает: сколько монет, успех рейда, поимка монстра, результат способности.

| Поле | Смысл |
|------|--------|
| coins | основная валюта |
| chaos | энергия за пакости |
| baseId | какая база закреплена за игроком |

| traps | установленные ловушки (позже) |

Стартовые значения MVP: 100 монет, 0 chaos, один Slime (Гуппи) при первом join.

---

## База игрока (особняк)

**Для игрока:** «моё логово» — спавн, лаборатория, сундук, ловушки, дверь.

**Для системы:** зона на карте с `BaseId`. При входе — первая свободная база, сохраняется в DataStore.

На карте: 6 баз (`Workspace.Bases.Base1`…`Base6`).

---

## Лаборатория (Phase 2+)

**Для игрока:** комната **на своём особняке**. Монстры стоят в **капсулах**. Здесь смотреть коллекцию, покупать и прокачивать (позже), выбирать кого отправить.

**Для системы:** зона, привязанная к `BaseId` игрока. ProximityPrompt или UI при входе. Не отдельная глобальная площадка на карте.

| Действие | Phase |
|----------|-------|
| Видеть монстров в капсулах | 2 |
| Покупка / прокачка | 6 |
| Кнопка «Отправить» → выбор цели | 3 |

**Без** тренировочной площадки, **без** обучающих NPC.

---

## Монстр

**Для игрока:** в капсуле — имя, редкость, состояние; в UI — «скрытность: высокая» (словами, не цифрами).

**Для системы:** тип (Slime, Gremlin…), статы, состояние:

| Состояние | Что видит игрок |
|-----------|-----------------|
| Idle | в капсуле, можно отправить |
| OnMission | «в пути» / «пакостит» |
| Fatigued | «отдыхает» |
| Captured | «пойман у игрока X» |

**Slime (Гуппи):** способность «липкая лужа» — срабатывает **на целевой базе** после прибытия монстра, не на тренплощадке.

---

## Отправка / миссия (Phase 3–4)

**Flow для игрока:**

1. В лаборатории выбрал монстра
2. Выбрал цель (база другого игрока или NPC-дом)
3. Монстр **сам идёт** к цели
4. На месте — пакость (лужа, саботаж…)
5. Награда **или** новость что монстра поймали

**Flow для системы:** сервер валидирует (монстр Idle, цель существует), запускает перемещение, по прибытии — способность, таймер, исход, fatigue, сохранение.

---

## Защита (Phase 5)

Владелец ставит ловушку и/или защитного монстра. Атакующий монстр может попасться → **клетка нарушителей** у владельца.
